# 12 - Execution Checklists

## Checklist por PR

1. Scope y plataformas afectadas identificadas.
2. Build/test OK en plataformas impactadas.
3. Contrato/API verificado si aplica.
4. Validacion UX nativa del cambio.
5. Riesgo residual documentado.

## Checklist por ola semanal

1. Objetivos de ola cerrados.
2. KPIs actualizados.
3. Riesgos reevaluados.
4. Gaps de paridad actualizados.

## Checklist por release candidate

1. Matriz de dispositivos completa aprobada.
2. Flujos core completos aprobados.
3. Cero P0/P1 abiertos.
4. Scorecard en verde.
5. Decisorio Go/No-Go firmado.

## Checklist de post-mortem (si hay regresion)

1. Causa raiz.
2. Impacto por plataforma.
3. Accion correctiva.
4. Guardrail agregado para que no se repita.
