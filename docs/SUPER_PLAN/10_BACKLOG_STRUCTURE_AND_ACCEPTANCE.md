# 10 - Backlog Structure and Acceptance

## Jerarquia

1. Epic (por ola).
2. Capability (cross-platform).
3. Story (por plataforma/dispositivo).
4. Task tecnica.

## Plantilla minima de Story

1. Objetivo de negocio.
2. Plataformas y dispositivos afectados.
3. Contrato/API impactado.
4. Criterios de aceptacion funcional.
5. Criterios de aceptacion UX nativa.
6. Criterios de prueba y evidencia.

## Definition of Done por Story

1. Build/test green en plataformas afectadas.
2. Checklist UX nativa aprobado.
3. Cero regresion en flujo core relacionado.
4. Documentacion actualizada si cambia contrato/flujo.

## Priorizacion

1. P0 estabilidad/contratos.
2. P1 flujo core y datos.
3. P2 optimizacion y mejoras no bloqueantes.
