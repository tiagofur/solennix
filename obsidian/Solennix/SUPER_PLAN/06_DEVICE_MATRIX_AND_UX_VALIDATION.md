# 06 - Device Matrix and UX Validation

## Matriz minima por release

1. Web desktop grande.
2. Web mobile (smartphone browser).
3. iPhone (compact).
4. iPad (portrait y landscape).
5. macOS (Designed for iPad) para validacion operativa.
6. Android smartphone.
7. Android tablet.

## Flujos obligatorios

1. Auth completo.
2. Crear evento completo.
3. Editar evento y reabrir resumen.
4. Registrar pago y validar consistencia.
5. Busqueda global y navegacion contextual.

## Criterios UX nativa

1. Jerarquia visual legible en cada viewport.
2. Patrones nativos respetados por plataforma.
3. Acciones principales visibles sin friccion.
4. Estados vacios/carga/error claros y accionables.

## Validacion accesibilidad

1. Contraste minimo WCAG.
2. Targets tactiles apropiados.
3. Lectores de pantalla y escalado de texto.
